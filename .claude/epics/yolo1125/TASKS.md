## 開發階段

### Phase 1: 基礎架構（Week 1）
```
001 → 002 ─┐
      ↓    │
      003 ─┘
```
- Task 001: 專案結構和資料庫（必須先完成）
- Task 002 & 003: 後端和前端基礎（可部分平行）

### Phase 2: AI 整合（Week 2）
```
004 ─┐
     ├→ 都依賴 001, 002
005 ─┘
```
- Task 004: YOLO 商品識別
- Task 005: 人臉識別登入
- **可平行開發**

### Phase 3: 業務邏輯（Week 2-3）
```
002,003,004,005 → 006 → 007
```
- Task 006: 購物車功能
- Task 007: 結帳流程

### Phase 4: 測試優化（Week 3）
```
001-007 → 008
```
- Task 008: 整合測試和優化

## 快速開始

### 1. 查看任務詳情
```bash
# 查看某個任務
cat .claude/epics/yolo1125/001.md
```

### 2. 開始開發
依照順序執行任務：
1. 先完成 Task 001（基礎）
2. 接著 Task 002 和 003（可同時開發）
3. 然後 Task 004 和 005（可同時開發）
4. 最後 Task 006 → 007 → 008（循序）

### 3. 追蹤進度
編輯 `epic.md` 中的任務清單，將 `[ ]` 改為 `[x]` 標記完成。
